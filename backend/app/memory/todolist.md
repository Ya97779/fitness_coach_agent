# 记忆模块待办事项

## 当前记忆架构

### 长期记忆 ✅（已实现）

| 类型 | 存储方式 | 说明 |
|------|---------|------|
| 用户画像 | SQLite `users` 表 | 身高、体重、年龄、性别、BMR、TDEE、目标体重、过敏史 |
| 每日统计 | SQLite `daily_logs` 表 | 当日摄入/消耗热量、食物记录、运动记录 |
| 历史汇总 | SQLite 数据汇总 | 近7天统计趋势、达标天数等 |

### 短期记忆 ✅（已实现）

| 类型 | 存储方式 | 说明 |
|------|---------|------|
| 当前对话 | LangGraph State | 仅在当前对话周期内有效 |
| 会话摘要 | 内存 | 超过10条消息时自动摘要，压缩早期对话 |

### 跨会话对话记忆 ❌（待实现）

| 类型 | 状态 | 说明 |
|------|------|------|
| 历史对话记录 | 未实现 | 之前与用户的对话内容未持久化 |
| 跨会话上下文 | 未实现 | 重启服务后 Agent 不记得之前的对话 |

## 待办事项

### P1 - 跨会话对话记忆（高优先级）

- [ ] **设计对话历史数据库表**
  - 创建 `conversation_logs` 表存储对话记录
  - 字段：id, user_id, agent_type, user_message, agent_response, created_at

- [ ] **实现对话历史持久化**
  - 在 `graph.py` 的 `process_user_message` 中保存对话
  - 每次交互后写入数据库

- [ ] **实现对话历史加载**
  - 用户发起新对话时，加载最近 N 条历史记录
  - 可配置加载条数（如最近20条）

- [ ] **实现对话要点提取**
  - 定期从对话中提取关键信息存入数据库
  - 例如：用户偏好、特殊需求、当前计划等

### P2 - 智能记忆管理（中优先级）

- [ ] **记忆优先级机制**
  - 短期：当前对话上下文
  - 中期：本週用户状态和偏好
  - 长期：用户画像和关键目标

- [ ] **记忆过期机制**
  - 自动清理过期的对话历史（如30天前）
  - 保留关键记忆长期存储

- [ ] **记忆压缩优化**
  - 对长对话进行智能摘要而非简单截断
  - 保留关键决策和建议

### P3 - 高级特性（低优先级）

- [ ] **多模态记忆**
  - 支持存储用户的图片（如饮食照片）
  - 支持语音输入的记忆同步

- [ ] **主动记忆**
  - Agent 主动询问确认用户偏好
  - 定期回顾用户目标达成情况
